--- layout: default title: 다이얼로그 ---

다이얼로그(dialog)는 앱에서 작업을 계속 진행하기 위해 결정사항이나 필요한 추가 정보를 사용자에게 묻습니다. 이런 요청들은 단순한 확인/취소 결정부터, 좀 더 복잡한 조절 설정이나 텍스트 입력까지의 범위에 듭니다.

  1. 선택 타이틀 영역

    타이틀(title)이 다이얼로그에 도입하였습니다. 예를 들자면 사용자가 바꾸려는 설정의 이름을 식별하거나 결정을 요구할 수 있습니다.

  2. 컨텐츠 영역

    다이얼로그의 내용은 다양하게 달라집니다. 설정 다이얼로그에서 앱이나 시스템의 설정을 사용자가 바꿀 수 있게 하는 슬라이더(slider), 텍스트 필드(text field), 체크박스(checkbox), 라디오 버튼(radio buttons)과 같은 사용자 인터페이스 요소를 포함합니다. 경고와 같은 다른 경우에는 사용자가 결정한 이후의 상황으로 진행하기 위한 단독적인 문구를 내용으로 포함합니다.

  3. 액션 버튼

    액션 버튼(action button)은 전형적으로 준비된 대부분의 동작과 같은 것들을 나타내는 취소 혹은 확인입니다. 그러나, 옵션들이 닫기, 확인 보다 대기, 내용에 설명한 동작 취소와 같은 특정 동작으로 이루어졌다면, 모든 buttons는 동작 단어가 될 것입니다. 이런 규칙에 따라, dialog의 무시 동작은 왼쪽에 두고, 긍정적인 동작 들은 오른쪽에 둡니다.

안드로이드에서의 전형적인 다이얼로그 사용 예시입니다.

경고

경고는 무언가를 진행하기 전에 확인이나 승인을 필요로 하는 상황을 사용자에게 알려줍니다. 이들은 심각성과 전달하는 메시지의 강조 모양새에 따라 약간 다릅니다.

타이틀 바 없는 경고

대부분의 경고는 제목이 필요 없습니다. 일반적인 결정들은 심각한 영향을 받지 않으며 하나 또는 두 가지의 문장으로 간결하게 요약할 수 있습니다. 내용 영역은 질문을 하거나("이 대화를 지우시겠습니까?" 와 같은 것들) 명백한 동작 버튼과 관련한 명확한 문장으로 구성합니다.

타이틀 바가 있는 경고

타이틀 바가 있는 경고는 드물게 사용합니다. 잠재적인 데이터, 연결 손실이나, 특별한 요금이 발생한 경우를 수반하는 매우 위험한 동작을 수행할 때만 적당하며, 명확한 질문이나 문장(타이틀) 그리고 약간의 추가적인 설명(내용 영역)이 필요합니다.

질문이나 문장을 간결하게 하세요. 예를 들면 "USB 저장소를 지우시겠습니까?"는 양해내용이 없습합니다. 사용자는 내용을 완전히 건너뛸 수 있어야 하고, 동작 버튼의 텍스트와 타이틀을 기반으로 어떤 선택을 할 수 있는지 명확한 의미를 알고 있어야 합니다.

팝업

팝업(popup)은 사용자로부터 단일 선택을 요구하는 다이얼로그의 가벼운 형태입니다. 팝업은 동작을 허용하거나 취소하는 명확한 버튼이 없습니다. 대신, 동작을 진행하기 전에 선택의 여지를 만들며, 팝업 바깥쪽을 터치하여 팝업을 없앨 수 있습니다.

토스트

토스트는 작은 팝업의 처리에 대해 가벼운 피드백을 제공합니다. 예를 들어, 여러분이 이메일을 보내기 전에 이메일 외부에서 탐색하면 여러분이 나중에 계속 편집할 수 있다는 것을 알려주는 "임시보관함" 토스트를 띄웁니다. 토스트는 시간이 지나면 자동으로 사라집니다.
