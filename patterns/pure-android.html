대부분의 개발자는 앱을 여러 플랫폼에 배포하고 싶어합니다. 안드로이드 앱을 준비할 때, 서로 다른 플랫폼은 서로 다른 규칙과 관례를 따른다는 것을 고려하십시오. 어느 한 플랫폼에서 완벽히 들어맞는 디자인이라도 다른 플랫폼의 문맥 상에서는 모양새와 느낌이 잘 맞지 않을 수 있습니다. "한번만 디자인하고, 어디나 배포하는" 방식이 당장은 시간을 절약해 줄지도 모르지만, 일관성이 떨어져 사용자가 어색하게 느끼는 앱이 되는 현실적인 문제를 감수하는 것입니다. 아래의 지침을 고려하여 흔히 빠질 수 있는 함정과 실수를 피해 가세요.

 

다른 플랫폼의 UI 요소를 흉내 내지 마세요

안드로이드 앱을 만드실 때, 다른 플랫폼의 테마를 가진 UI 요소를 그대로 가져오거나 흉내내려 하지 마세요. 이 스타일 안내서의 Building Blocks 부분을 검토하여, 안드로이드의 주요 UI 요소에는 어떤 것이 있고 시스템 기본 테마에서 어떤 모양인지 알아보세요. 또한 안드로이드 플랫폼에 제공되는 앱을 참고해서 그러한 요소가 앱에 어떤 방식으로 적용되어 있는지 감을 잡으세요. UI 요소를 입맛에 맞게 고치실 때는 다른 플랫폼의 관례를 따르는 것이 아니라, 여러분의 브랜딩에 맞추어 조심스럽게 해야 합니다.

안드로이드와 iOS, 윈도우 폰 7의 UI 요소.
 

플랫폼 제공 아이콘을 그대로 가져오지 마세요

각 플랫폼마다 공유, 새 문서나 삭제 등 자주 사용되는 중요 아이콘을 제공합니다.

안드로이드로 앱을 이식할 때에는 그러한 플랫폼 아이콘을 그에 대응하는 안드로이드 아이콘으로 바꾸세요.

앱에 사용할 수 있는 다양한 아이콘을 안드로이드 SDK에서 찾을 수 있습니다.

안드로이드와 iOS, 윈도우 폰 7의 아이콘.
 

탭 바를 화면 하단에 두지 마세요

다른 플랫폼에서는 앱의 여러 뷰 사이를 전환하기 위해 하단 탭 바를 사용합니다. 안드로이드는 탭을 화면 위쪽 액션 바에 표시하는 것이 관례입니다. 추가로, 분리된 액션 바에 액션을 표시하기 위해 하단 바를 사용할 수도 있습니다.

안드로이드 플랫폼의 다른 앱들과 일관성 있는 경험을 위해, 또한 뷰 전환과 액션을 혼동하지 않도록 하기 위해서 반드시 이 지침을 따르셔야 합니다.

뷰 전환을 위해 액션 바를 올바르게 사용하는 방법은 액션 바에서 보실 수 있습니다.

액션 바에 탭이 있는 안드로이드 전화 걸기와 iOS 하단 탭 비교.

액션 바의 뒤로 가기 버튼에 레이블을 달지 마세요

다른 플랫폼에서는 사용자가 애플리케이션의 상위 계층으로 이동할 수 있도록 하기 위해 레이블이 붙은 명시적인 뒤로 가기 버튼을 사용합니다. 안드로이드에서는 그와 달리, 상위 계층으로 이동할 때는 주 액션 바의 앱 아이콘을, 일시적인 이동을 할 때는 네비게이션 바의 뒤로 가기 버튼을 사용합니다. 자세한 내용은 네비게이션 패턴을 참고하세요.

플랫폼 전반에서 일관성 있는 네비게이션 경험을 제공할 수 있도록 이 지침을 따르십시오.

상위 표시가 달린 안드로이드 액션 바와 레이블이 붙은 iOS "뒤로 가기" 버튼 비교.

줄 항목 끝에 오른쪽 표시를 달지 마세요

다른 플랫폼에서는 추가 내용을 볼 수 있는 경우 줄 항목 끝에 오른쪽 표시를 다는 패턴이 자주 사용됩니다.

안드로이드에서는 드릴 다운 줄 항목에 그러한 표식을 사용하지 않습니다. 플랫폼과의 일관성을 유지하도록, 또한 사용자가 그 표식이 무엇을 뜻하는 지 추측할 필요 없도록 사용을 피하세요.

줄 항목 끝에 오른쪽 표시가 없는 안드로이드 설정 화면과 iOS 설정 화면 비교.

장치 독립성

여러분의 앱이 여러 가지 서로 다른 크기의 화면에 표시된다는 것을 잊지 마세요. 다른 크기와 밀도를 가진 화면에 맞도록 비주얼 애셋을 만들고, 장치 형태에 따라 UI 크기가 적절하게 조정될 수 있도록 multi-pane layout 등의 개념을 적용하십시오.

자세한 내용은 이 디자인 지침서의 Devices and Displays와 Multi-pane Layouts를 참고하세요.
