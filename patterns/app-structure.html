앱은 여러 요구 사항에 따라 다양한 형태가 됩니다. 예를 들어:

  • 계산기나 카메라와 같은 앱은 하나의 스크린에서 조작되는 하나의 집중된 액티비티로 만들어 집니다.
  • 휴대전화와 같은 앱은 더 깊이 내려가지 않고 동일한 깊이의 다른 액티비티 사이를 이동하는 용법을 가집니다.
  • Gmail과 Google Play와 같은 앱은 깊이 들어가는 데이터 뷰의 큰 집합으로 구성되어 있습니다.

당신의 앱의 구조는 컨텐츠와 사용자가 사용하길 원하는 작업에 크게 달려있습니다.

일반 구조

일반적인 안드로이드 앱은 최상위 수준의 뷰와 구체/편집 뷰로 구성됩니다. 탐색 계층(navigation hierachy)이 깊고 복잡해지면 카테고리 뷰가 최상위 수준과 세부 뷰들을 연결합니다.

최상위 수준 뷰

앱의 최상위 수준 뷰는 전형적으로 앱의 지원에 따라 다른 뷰를 가집니다. 이 뷰는 같은 데이터에 대해 다른 형태로 표현것과 앱의 전적으로 다른 기능적인 면을 모두 보여주고 있습니다.

 

카테고리 뷰

카테고리 뷰는 당신의 데이터로 더 깊이 내려가게 합니다.

 

세부/편집 뷰

세부/편집 뷰는 데이터를 소모하거나 만드는 곳입니다.

최상위 수준 뷰

시작 화면의 레이아웃은 특별한 관심이 필요합니다. 시작 화면은 앱이 수행된 후 사용자가 보는 첫 화면입니다. 그렇기 때문에 시작 화면은 새로운 사용자와 기존 이용자 모두에게 좋은 사용자 경험을 주어야 합니다.

자신에게 물어보세요: "보통 유저는 내 앱에서 뭘 하길 원할까요?" 그리고 거기에 맞추어 첫 화면의 사용자 경험을 구성하세요.

컨텐츠를 전면에

많은 앱들이 컨텐츠 표시에 집중합니다. 탐색만 하는 화면을 만들지 말고 시작 화면 중앙에 컨텐츠를 두어 사용자가 앱의 알맹이를 바로 가져가게 하세요. 데이터 타입과 화면 크기에 적합하고 시각적으로 두각시킬 수 있는 레이아웃을 선택하세요.

Google Play 앱의 시작 화면은 먼저 앱, 음악, 책, 영화, 게임 상점을 탐색할 수 있게 합니다. 사용자에게 흥미로운 컨텐츠가 나열된 잘 정리된 추천과 홍보물이 가득합니다. 검색은 액션 바를 통해 보기좋게 쓸 수 있습니다.

탐색과 액션을 위한 액션 바 설정

앱의 모든 화면은 일관된 탐색과 중요한 액션의 표시를 위해 액션 바를 표시해야 합니다.

최상위 수준에서 엑션바를 적용하는데 특별히 고려해야할 점이 있습니다:

  • 액션 바에 앱의 아이콘이나 제목을 사용합니다.
  • 만약 최상위 수준에서 여러 뷰를 포함하거나 다른 사용자 계정의 데이터 사이로 이동한다면, 뷰 사이의 이동이 사용자에게 명확하게 보이도록 액션 바에 뷰 컨트롤을 붙이세요.
  • 앱에서 사용자가 컨텐츠를 만들어 낸다면 최상위 수준에서 컨텐츠를 바로 접근할 수 있게 하세요.
  • 만약 컨텐츠가 검색가능하다면 액션 바에 검색 액션을 추가하여 사용자가 탐색 계층 사이로 갈 수 있게 하세요.
이메일은 생산성과 관련됩니다. 그래서 효율적인, 데이터가 잘 밀집되어 동작하는 리스트 씁니다. 탐색은 계정과 최근 레이블 사이를 이동하게 합니다. 편지쓰기와 검색 아이콘은 하단의 분할 액션 바에 잘 보입니다.

앱의 정체성 만들기

액션 바를 넘어 앱의 정체성을 만듭시다. 당신 앱의 정체성은 데이터와 데이터가 구성되는 방법, 사용자와 상호작용하는 방식에 따라 결정됩니다. 특히, 미디어 위주의 애플리케이션에서는 단순한 리스트 뷰의 단조로움을 넘어서 데이터를 뽐내는 독특한 레이아웃을 시도해 보세요.

3D 회전목마(carousel) 커버 아트를 기리고 뮤직 앱을 독창적으로 만들었습니다. 최근 뷰는 기본적으로 사용자가 마지막에 들었던 음악을 맞춰줍니다.

카테고리 뷰

깊이 있는 데이터 위주의 앱은 보통 조직화된 카테고리를 통해 데이터를 보이고 다루는 상세 뷰를 탐색합니다. 카테고리는 앱을 피상적으로 유지하여 탐색 단계를 인지하는 노력을 최소화 시킵니다.

치솽위 수준에서 상세 뷰까지의 수직 탐색 단계는 보통 앱의 컨텐츠 구조에 의해 영향을 받지만 탐색을 인지하는 걸 부담스럽게 하는 부분들을 제거할 방법은 여럿 있습니다.

선택과 데이터 표현을 통합하기 위해 탭을 사용하세요.

카테고리들이 유사하고 한정적이다면 탭의 사용은 성공적일 겁니다. 계층의 수준을 제거하고 데이터에 사용자가 집중하게 하는 것은 이점이 있습니다. 데이터가 많은 카테고리들을 좌우로 이동하며 탐색하는 것은 명시적인 탐색 단계 보다 일반적인 브라우즈 경험과 흡사합니다.

 

만약 카테고리가 유사하고, 예측가능하고, 꽤 밀접하다면, 스크롤 탭(Scrolling tabs)를 사용하세요. 탭의 일부 항목만 목록에 표시됩니다. 관리가능한 수준에서 스크롤 탭의 수를 유지하는 것은 탐색이 덜 힘들게 합니다. 중요한 규칙이 있습니다: 5–7 탭 이상은 안됩니다.

Google Play는 탭을 사용하여 카테고리 선택과 컨텐츠를 동시에 보여줍니다. 카테고리 사이의 탐색을 위해 사용자는 컨텐츠를 왼쪽이나 오른쪽으로 스와이프할 수 있습니다.

탭 내의 카테고리가 밀접하지 않다면 고정 탭을 선호합니다. 모든 카테고리가 탭의 항목으로 보입니다.

YouTube는 상대적으로 기능 부분의 연관없는 다른 뷰로 이동하기 위해 고정 탭을 사용합니다.

계층을 가로 지르기

목표에 빨리 도달하기 위해 지름길을 허용하는 것은 이점이 있습니다. 리스트나 그리드 뷰의 데이터 항목에서 액션을 바로 수행하기 위해 드롭다운이나 분할된 리스트 아이템으로 보여줍니다. 이 방법은 계층 아래로 탐색할 필요없이 데이터에서 액션을 수행할 수 있게 합니다.

Music은 사용자가 카테고리 뷰 (앨범)에서 데이터 항목 (노래)로 직접 접근할 수 있기 때문에, 노래 상세 뷰를 탐색할 필요가 없습니다.

여러 데이터 항목에 대해 행동하기

카테고리 뷰가 대부분 사람들을 상세적인 컨텐츠로 안내하지만, 가끔 데이터 모음으로서 행동하는 것도 이점이 있다는 것을 명심하십시요.

예를 들어, 상세 뷰에서 항목을 삭제하는 것을 허용하면 카테고리 뷰에서 여러 아이템을 삭제하는 것도 허용해야 합니다. 개별 뷰의 행동이 항목의 모음에서 어떻게 응용될 수 있을지 분석하세요. 카테고리 뷰에서 다중 항목에 대한 행동을 할 수 있게 다중 선택을 사용합시다.

상세 뷰

상세 뷰는 사용자의 데이터를 보고 행동하게 합니다. 상세 뷰의 레이아웃은 표현될 데이터에 달려있으며 앱에 따라 다양합니다.

레이아웃

상세 뷰에서 동작시키고 조작할 엑티비티와 이용할 레이아웃에 따라 엑티비티들을 고려하세요. 전체 화면으로 정신 사납지 않으면서 실감나게 하려면면 소등된 것 (기타 UI가 최소화 된 버전) 을 만들어야 합니다.

Google Book의 상세 뷰는 실제 책의 독서 방식을 모사하였습니다. 페이지 넘기는 에니메이션이 그 개념을 확실히 하죠. 실감나는 사용자 경험을 만들기 위해 앱은 시스템의 UI 어포던스를 모두 제거하는 소등을 했습니다.
피플 앱의 상세 뷰의 목적은 커뮤니케이션 옵션을 보여주는 것입니다. 리스트 뷰는 효과적으로 훑어보고 휴대전화 번호, 이메일 주소, 다른 항목등을 쉽게 접근하게 합니다. 항목 하나가 전화 걸기와 문자 보내기 등으로 나뉘어진 항목으로 사용되기도 합니다.

상세 뷰들을 효과적으로 탐색하기

사용자가 순차적으로 여러 아이템을 보길 원할 것 같으면 사용자에게 상세 뷰 들을 탐색할 수 있게 하세요. 그렇게 하기 위해 뷰 스와이프나, 다른 방법(필름 스트립) 등을 도입하세요.

Gmail은 상세 뷰에서 다른 상세 뷰로 탐색하기 위해 뷰 스와이프를 사용합니다.
갤러리는 이미지 들을 왼쪽이나 오른쪽으로 이동하는 스와이프 제스처를 제공하는 것 이외에 필름 스트립 컨트롤을 제공합니다. 필름 스트립 컨트롤에서는 특정 이미지로 바로 이동할 수 있습니다.

체크리스트

  • 화면에 유용한 컨텐츠를 표현할 방법을 찾으세요.

  • 일관된 탐색을 위해 액션 바를 사용하세요.

  • 수평적인 탐색과 숏컷을 사용하여 계층을 얕게 유지하세요.

  • 데이터의 묶음에 사용자가 명령할 수 있게 다중 선택을 지원하세요.

  • 여러 상세 아이템 사이를 빨리 탐색하기 위해 스와이프 뷰 등을 사용하세요.
