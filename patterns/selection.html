안드로이드 3.0에서 길게누르기(같은 위치를 잠시 누르는 터치) 제스쳐는 데이터를 선택하는 일반적인 제스쳐로 소개되었습니다. 이것은 여러분 앱의 다중 선택과 컨텍스트 액션 처리 방식에 영향을 주었습니다.

 

무엇이 바뀌었는가?

이전버전의 안드로이드에서, 롱 프레스 제스쳐는 선택된 아이템의 문맥 액션을 문맥 메뉴에 표시하기위해 널리 사용되었습니다.

이 패턴은 안드로이드 3.0에서 바뀌었습니다. 길게누르기 제스쳐는 이제 문맥 액션 바(CAB, contextual action bar)로 불리우는 새로운 요소에 들어있는 데이터 선택, 문맥 액션, 선택 관리 기능을 위해 쓰여집니다.

문맥 메뉴를 보여주기위한 길게누르기 제스쳐의 전형적인 사용.

문맥 액션 바 (CAB)의 사용

선택 문맥 액션 바는 데이터가 선택되어있는 동안 여러분의 현재 액션바 위에 중첩되는 임시 액션바 입니다. CAB는 사용자가 선택 가능한 데이터를 롱 프레스 한 다음에 표시됩니다.

 

여기서 사용자가 할 수 있는 것:

  • 아이템을 터치해서 추가적으로 아이템을 선택하세요.
  • 하이라이트된 모든 데이터 아이템들에게 문맥 액션 바에 있는 액션을 수행하세요. 이후 문맥 액션 바는 자동적으로 사라집니다.
  • Back 버튼 또는 문맥 액션 바의 체크마크 버튼으로 문맥 액션 바를 사라지게 하세요. 선택되었던 모든 하이라이트들도 문맥 액션 바와 함께 사라집니다.

문맥 액션 바의 액션들 선택하기

여러분들은 문맥 액션 바에 표시할 액션과 요소들을 결정 할 수 있습니다. 어느 아이템이 꼭대기에 위치할지,액션 오버플로우에 위치할지를 결정하기위해 액션 바 패턴 가이드라인을 사용하세요.

문맥 액션 바의 액션을 동적으로 변경하기

사용자가 선택에 아이템을 더 추가하는 대부분의 경우, 여러분들은 문맥 액션 방의 액션들을 동적으로 변경해야만 합니다. 하나만 선택된 데이터 아이템에 적용하는 액션들이 다중 선택된 같은 종류의 데이터 아이템에 적용되지는 없습니다.

추가 아이템이 선택된 경우에 문맥 액션 바의 액션들이 변경됨.

체크리스트

  • 여러분의 앱이 다중 데이터 아이템 선택을 지원할 때마다, 문맥 액션 바(CAB)를 사용하십시요.

  • 롱 프레스 제스쳐를 배타적으로 남겨두세요. 전형적인 문맥 메뉴를 표시하는 방법으로 사용하지 마세요.

  • 만약 리스트에서 다중 선택을 지원하지 않는 다면, 롱 프레스는 아무 것도 하지않게 두세요.

  • 여러분들이 여러분의 액션바에 넣으려고 계획했던 액션들과 같은 방식으로 문맥 액션 바에 표시하고 싶은 액션들을 계획하세요.
