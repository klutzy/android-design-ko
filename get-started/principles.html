--- layout: default title: 디자인 원칙 ---

이 디자인 원칙은 안드로이드 사용자 경험 팀에서 제작하였으며, 사용자의 쾌적한 편의를 위해 구성했습니다. 이 원칙을 여러분만의 창의성과 디자인 구상에 적용해 보고, 필요하면 상황에 맞게 수정하세요.

마법을 걸어 주세요

놀라운 방법으로 기쁨을 주세요

아름다운 배경, 잘 배치된 애니메이션, 상황에 딱 맞는 효과음을 넣으면 사용하기 즐거워집니다. 작은 효과를 추가하는 것만으로도 사람들은 쉽고 편리하며 강력한 기능이 있다고 느낍니다.

 

버튼이나 메뉴보다는 실물이 낫죠

앱 화면에 나오는 대상들을 손으로 바로 고칠 수 있도록 만드세요. 인지 부하도 줄어들고, 더 재미있습니다.

 

내 것으로 만들 수 있도록 해주세요

사람들은 프로그램에 사용자 개인 취향을 넣고 싶어합니다. 자기 집에 온 것처럼 편안함과 통제력을 느끼거든요. 멋진 기본 환경을 제공하는 것도 중요하지만, 재미있는 개인 맞춤 설정을 넣어 보세요. 물론 원래의 기능을 가리지는 않도록요.

 

나를 알아가 주세요

사람들의 취향을 점차 배우세요. 사용자에게 같은 질문을 자꾸 하는 대신, 쉽게 이전의 선택을 할 수 있도록 기억해 두세요.

생활을 편리하게 해주세요.

간단히

단어는 쉽게 문장은 짧게. 사람들은 긴 문장을 잘 읽지 않습니다.

 

백문이 불여일견

글 대신 그림으로 설명해 보세요. 사용자의 주의를 끌기에도 좋고 훨씬 효율적이기도 합니다.

 

대신 판단하고 마지막에만 물어보세요

무조건 사용자에게 묻기보다는 최선의 선택을 추측해서 실행해보세요. 선택지가 너무 많으면 사람들은 불편해합니다. 추측이 틀렸다면 '되돌리기'만 할 수 있으면 됩니다.

 

필요할 때 필요한 것만 보여주세요

사람들은 한번에 너무 많이 보면 어리둥절해 합니다. 기능이나 정보를 한눈에 볼 만큼 작게 쪼개주세요. 당장 중요하지 않은 기능은 숨기고, 필요할 때 사용자에게 알려주세요.

 

내가 어디에 있는지 알아야죠

사용자가 지금 어떤 화면에 있는지를 알 수 있어야 합니다. 앱의 맥락에 맞춰 다른 화면이 나오도록 만들고, 화면 전환을 통해 여러 화면이 어떻게 관련있는지 표현하세요. 작업 진행에 따라 알맞은 응답도 해 주세요.

 

내 자료 날아가면 안돼요

자료 생성 시각을 기록하고 어디서나 접근할 수 있도록 하세요. 개인 설정이나 핸드폰/태블릿/컴퓨터 등에서 만든 자료를 저장해 두세요. 기기 업그레이드는 일도 아니게 됩니다.

 

똑같아 보이면 똑같이 행동해야죠

서로 다른 기능을 겉보기에도 다르게 만들면 기능을 구별하는 데 큰 도움이 됩니다. 겉보기만 비슷해 보이고 같은 조작에 다르게 행동하는 것을 피하세요.

 

정말 중요할 때만 날 찾으세요

유능한 비서처럼 자질구레한 일로 사용자를 방해하지 마세요. 사람들은 지금 하는 일에 집중하고 싶어합니다. 정말 중요하고 급한 일이 아니라면 귀찮아요.

놀라운 경험을 만들어주세요

만능 비법을 주세요

스스로 깨닫고 이해하는 것만큼 즐거운 일도 없습니다. UI 패턴이나 다른 안드로이드 앱에서 사용하는 방법들을 적용해보세요. 여러분의 앱이 더 배우기 쉬워집니다. 가령, 스와이프 제스처는 탐색을 쉽게 만들어주는 패턴입니다.

 

내 잘못 아니에요

사용자 입력을 고쳐받을 때는 부드럽게 응답하세요. 사람들은 여러분의 앱을 똑똑하게 쓰고 싶어합니다. 뭔가 잘못되었다면 어떻게 되돌릴 수 있는지 이해하기 쉽게 설명해주세요. 기술적인 사항은 빼고요. 앱이 알아서 고칠 수 있다면 훨씬 좋겠죠.

 

자주 칭찬해주세요

복잡한 작업을 쉽게 해낼 수 있는 여러 개의 단계로 나누어 주세요. 사용자의 행동에는 간단한 것도 응답을 주세요.

 

대신 조종해주세요

초보자는 상상도 못 했을 기능을 넣어서, 사람들에게 전문가가 된 기분을 전해주세요. 예를 들어, 여러 사진 효과를 하나로 묶어주면 누구나 쉽게 멋진 사진을 찍을 수 있습니다.

 

중요한 것을 간편하게 만드세요

모든 기능이 중요하지는 않습니다. 앱에서 가장 중요한 기능을 골라 쉽게 찾아 빠르게 쓸 수 있도록 만드세요. 카메라 셔터 버튼이나 음악 재생기의 일시정지 기능처럼요.
