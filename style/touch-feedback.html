--- layout: default title: 터치 피드백 ---

색과 빛을 활용해 터치에 반응하고 제스처 결과를 강조하고 가능하거나 불가능한 액션을 표시하세요.

사용자가 액션이 있는 영역을 터치했을 때는 언제나 시각적으로 반응하세요. 사용자는 이 반응으로 자신이 어떤 객체를 터치했는지 그리고 앱이 "입력을 기다리는지(listening)" 알 수 있습니다.

상태

 
대부분의 안드로이드 UI 요소는 터치 상태를 알려주는 효과 등의 터치 피드백을 지니고 있습니다.
 

커뮤니케이션

객체가 더 복잡한 제스쳐에 반응한다면 사용자는 작업 결과를 더 쉽게 예상합니다. 예를 들어 최근 앱에서 섬네일을 왼쪽이나 오른쪽로 스와이프하기 시작하면 섬네일도 희미해지기 시작합니다. 사용자는 이 반응으로 스와이프를 하면 아이템이 사라진다고 예상합니다.

 

경계(Boundaries)

사용자가 스크롤 가능한 영역의 위나 아래 경계 밖으로 스크롤하려 할 때 시각 신호로 경계라는 것을 전달할 수 있습니다. 예를 들어 사용자가 첫번째 홈 화면 패널에서 더 끝으로 스크롤하면 화면의 컨텐츠는 오른쪽으로 기울어져 이 방향으로 더 이동할 수 없음을 보여줍니다. 안드로이드의 대부분의 스크롤 가능한 UI 위젯은(리스트, 그리드 리스트 등) 이미 경계 피드백을 내장하고 있습니다. 만약 커스텀 위젯을 만들고 있다면, 경계 피드백을 잊지 말고 앱에서 제공해주세요.
